upstart 설정

/etc/init/appname.conf

  
description &quot;appname&quot;
  
author &quot;xxx&quot;

start on startup
  
stop on shutdown

chdir /home/xxx/appname
  
respawn

script
      
export HOME=&quot;/home/xxx&quot;
      
export AWS\_ACCESS\_KEY_ID=
      
export AWS\_SECRET\_ACCESS_KEY=
      
exec /home/xxx/appname/app -c=&quot;config.json&quot;
  
end script
  

  
$ sudo initctl reload-configuration
  

  
$ sudo start appname

AWS Credentials가 upstart 등에서 제대로 동작하지 않을 경우

~/.aws/credentials 파일에 AWS의 Credentials Key를 설정해 놓고 개발하다가,

upstart나 superviosord등을 사용해 실행하게 되면 다음과 같은 에러가 발생한다.

  
NoCredentialProviders: no valid providers in chain. Deprecated.

이럴 경우 다음과 같이 AWS Credentials 정보를 환경변수로 설정해서 사용하면 문제 없이 사용 가능하다.

관련 정보는 링크를 참고한다.
